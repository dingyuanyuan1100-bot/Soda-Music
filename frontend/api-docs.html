REMOTE API DOCUMENTATION

Soda Music Third-Party Interface Docs

This frontend is served as a companion to the remote API service running behind http://47.116.42.11:1234. The repository is for a remote-connected project, not a local-only product.

Back To Workspace Markdown Swagger Routes JSON

GET/api/health

Health check for the remote API service and its upstream connectivity.

GET/api/routes

Returns the currently exposed route list for the deployed service.

GET/api/search

Unified search route.

Parameters: q=keywords&type=songs|playlists

GET/api/search/songs

Search songs through the remote API layer.

Parameters: q=keywords

GET/api/search/playlists

Search playlists. The service may enrich some playlist covers automatically.

Parameters: q=keywords

GET/api/playlist/share-meta

Fetch share-page metadata for a playlist by ID.

Parameters: id=playlistId

GET/api/playlist/detail

Fetch playlist detail and track list.

Parameters: id=playlistId

GET/api/song/detail

Fetch detail for a single shared song entry.

Parameters: id=songId

GET/api/song/full

Aggregated song route for detail, lyrics, and playability data.

Parameters: id=songId&name=title&artist=artist&album=album&cover=coverUrl

GET/api/lyrics

Fetch lyrics, preferring upstream responses and falling back when needed.

Parameters: id=songId&name=title&artist=artist&album=album

GET/api/playback/check

Check whether a song appears playable and what source is preferred.

Parameters: id=songId&name=title&artist=artist&album=album&cover=coverUrl

GET/api/stream

Audio stream proxy route designed for direct use by an audio player.

Parameters: id=songId&name=title&artist=artist&album=album&cover=coverUrl
Optional: preferred=share&shareUrl=...

GET/api/cover

Image proxy route with fallback handling.

Parameters: url=imageUrl

GET/api/auth/status / probe / pages / my-playlists-probe

Auth-debugging routes used to inspect client-side login signals for the remote-connected workflow.
